# Summarization Rules

## 中文总结写作规则

1. 先写研究对象或问题。
2. 再写方法或技术路线。
3. 最后写适合什么场景阅读。
4. 不使用夸大性词语。
5. 不编造实验结果、对比结论、性能数字。

## 信息来源边界

总结只能依据：

1. 论文标题。
2. 公开摘要。
3. venue、日期、作者等元数据。
4. Semantic Scholar 的 fieldsOfStudy / citationCount / OA PDF 元数据。
5. 本地关键词命中情况。

## 摘要来源优先级

统一 `abstract` 字段应按以下优先级产生：

1. `semantic_scholar_abstract`
2. `crossref_abstract`
3. existing abstract

报告中应显式展示 `abstract_source`，避免黑箱替换。

## 摘要为空时如何处理

1. 明确写明未获取到公开摘要。
2. 总结退化为基于标题和关键词的保守描述。
3. 不推断模型结构、实验设置和结果。

## PDF 链接说明

1. `openAccessPdf.url` 只是公开元数据中的链接信息。
2. 有 PDF 链接不代表一定可访问、长期有效或可直接下载。
3. 报告中要把它表述为“可进一步获取/核对原文的线索”，而不是“已下载 PDF”。

## 推荐理由写法

推荐理由必须透明、可解释。常用模板：

1. 发表时间较新，适合快速了解近期方向。
2. 同时命中多个关键词，适合交叉主题阅读。
3. 摘要较完整，便于初筛。
4. DOI、链接和 venue 信息完整，便于继续追踪。
5. 存在开放获取 PDF 链接，可进一步核对原文。
6. citationCount 具备一定参考价值，但只作为弱辅助信号。

## 多 venue 场景注意事项

1. 不要把多 venue 检索结果硬写成某一个固定期刊或会议的结论。
2. 总览段落要明确写出实际来源范围。
3. 如果某次联合检索最终只命中其中一个 venue，也应如实写明，不夸大覆盖面。
